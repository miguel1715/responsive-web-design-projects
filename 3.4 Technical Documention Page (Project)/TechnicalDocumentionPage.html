Selectors

Selectors tell CSS which elements to style. They allow you to target elements by their type, class, id, attributes, and relationships inside the HTML structure.

Choosing the right selector makes your CSS cleaner, more reusable, and easier to maintain as your project grows.

  • Selector — targets elements by their tag name (e.g., p, h1, img)
    • p { color: blue; }
  • Class selector — targets elements with a class using a dot (.container)
    • .btn { padding: 1rem; }
  • ID selector — targets a unique element using a hashtag (#main)
    • #main-title { font-size: 2rem; }
  • Attribute selector — targets by attributes (e.g., input[type="text"])
    • input[type="text"] { border: 1px solid gray; }
  • Pseudo-classes — target states (:hover, :focus, :active)
    • a:hover { text-decoration: underline; }
  • Pseudo-elements — target parts of elements (::before, ::after)
    • p::before { content: "→ "; color: gray; }
  • Combinators — target elements based on relationships (descendants, siblings)
    • .nav > li { margin-bottom: 0.5rem; }
Box Model

The CSS Box Model describes how every element on a webpage is structured. Each element is treated as a box made up of content, padding, border, and margin.

Understanding the box model is essential for controlling spacing, layout, and alignment.

  • Box-sizing — controls whether padding/border count in the width
    • * { box-sizing: border-box; }
  • Content — the actual text or image inside the element
    • p { width: 200px; height: 100px; }
  • Padding — space between the content and the border
    • div { padding: 20px; }
  • Border — the line surrounding the padding and content
    • img { border: 2px solid black; }
  • Margin — space outside the border between elements
    • .card { margin: 1.5rem; }
Positioning

CSS positioning controls how elements are placed on the page. It allows you to move elements, stack them, or remove them from the normal document flow.

Understanding positioning is essential for layouts, navigation bars, modals, tooltips, and advanced UI patterns.

  • Static — the default positioning; elements follow normal flow
    • div { position: static; }
  • Relative — element stays in normal flow but can be shifted using top/left/right/bottom
    • .box { position: relative; top: 10px; left: 20px; }
  • Absolute — removed from normal flow and positioned relative to the nearest non-static ancestor
    • .child { position: absolute; top: 0; right: 0; }
  • Fixed — positioned relative to the viewport; does not move when scrolling
    • nav { position: fixed; top: 0; left: 0; }
  • Sticky — scrolls normally until it reaches a threshold, then sticks in place
    • header { position: sticky; top: 0; }
  • z-index — controls stacking order (which elements appear on top)
    • .modal { z-index: 10; position: absolute; }
Flexbox

Flexbox is a one-dimensional layout system that makes it easy to align, distribute, and space elements inside a container. It is especially useful for responsive layouts.

Flexbox works along two axes: the main axis and the cross axis. You control these with direction, alignment, and distribution properties.

  • Display: flex — turns an element into a flex container
    • .container { display: flex; }
  • Flex-direction — sets the direction of items (row, column)
    • .container { flex-direction: row; }
  • Justify-content — aligns items along the main axis
    • .container { justify-content: center; }
  • Align-items — aligns items along the cross axis
    • .container { align-items: flex-start; }
  • Flex-wrap — allows items to wrap onto multiple lines
    • .container { flex-wrap: wrap; }
  • Gap — controls spacing between flex items
    • .container { gap: 1rem; }
Typography

Typography in CSS controls how text appears on the page—its size, weight, spacing, and readability. Good typography improves accessibility and the overall user experience.

CSS gives you fine control over fonts, line spacing, letter spacing, and how text flows within the layout.

  • Font-family — sets the typeface used for text
    • body { font-family: "Inter", sans-serif; }
  • Font-size — controls how large the text is
    • h1 { font-size: 2.5rem; }
  • Font-weight — adjusts boldness
    • strong { font-weight: 700; }
  • Line-height — sets spacing between lines of text
    • p { line-height: 1.6; }
  • Letter-spacing — adjusts spacing between characters
    • .title { letter-spacing: 0.05rem; }
  • Text-align — aligns text (left, center, right)
    • .center { text-align: center; }
Colors & Backgrounds

CSS provides multiple ways to control color and backgrounds, allowing you to set mood, hierarchy, and visual structure in your UI. You can use named colors, HEX, RGB, HSL, gradients, and images.

Background styles can apply to any element and are essential for layout, emphasis, and branding.

  • Color — sets the text color
    • p { color: #333; }
  • Background-color — sets the background color of an element
    • div { background-color: lightblue; }
  • Background-image — adds images or gradients
    • header { background-image: url("banner.webp"); }
  • Background-size — defines how images scale
    • div { background-size: cover; }
  • Background-repeat — controls if images repeat
    • section { background-repeat: no-repeat; }
  • Background-position — positions background images
    • div { background-position: center; }
Grid

CSS Grid is a powerful layout system that allows you to create two-dimensional layouts—both rows and columns. It is ideal for building full-page layouts, dashboards, galleries, and complex structures.

  • Display: grid — enables grid layout
    • .container { display: grid; }
  • Grid-template-columns — defines column structure
    • .container { grid-template-columns: 1fr 1fr 1fr; }
  • Grid-template-rows — defines row structure
    • .container { grid-template-rows: auto 200px; }
  • Gap — sets spacing between grid items
    • .container { gap: 1rem; }
  • Grid-column / grid-row — controls how elements span
    • .item { grid-column: span 2; }
  • Place-items — aligns items in both directions
    • .container { place-items: center; }
Units and Sizing

CSS provides various units to size elements flexibly. Choosing the right unit helps create responsive, scalable designs.

  • px — fixed pixel values
    • div { width: 300px; }
  • % — relative to the parent element
    • img { width: 50%; }
  • em — relative to the element’s font size
    • p { margin-left: 2em; }
  • rem — relative to the root font size
    • h1 { font-size: 2rem; }
  • vw / vh — percentage of viewport width/height
    • .hero { height: 100vh; }
  • min(), max(), clamp() — powerful fluid sizing functions
    • p { font-size: clamp(1rem, 2vw, 2rem); }
Transitions & Transforms

Transitions allow smooth changes between property values, while transforms let you rotate, scale, move, or skew elements.

  • Transition — animates property changes
    • button { transition: background 0.3s ease; }
  • Transform: translate() — moves an element
    • .box { transform: translateX(20px); }
  • Transform: rotate() — rotates an element
    • .icon { transform: rotate(45deg); }
  • Transform: scale() — resizes an element
    • img:hover { transform: scale(1.1); }
  • Transform: skew() — skews an element
    • .title { transform: skewX(10deg); }
  • Transition-property / duration / timing-function — fine-tune animations
    • .link { transition: color 0.2s linear; }
Responsive Design

Responsive design ensures your layout adapts to different screen sizes, from desktops to mobile devices. CSS provides multiple tools to make designs flexible and accessible.

  • media queries — apply CSS only at specific breakpoints
    • @media (max-width: 600px) { body { font-size: 14px; } }
  • flexbox — naturally responsive layout tool
    • .container { display: flex; flex-wrap: wrap; }
  • grid — powerful for adaptive layouts
    • .grid { grid-template-columns: repeat(auto-fit, minmax(200px, 1fr)); }
  • fluid units — % , vw , vh , rem for scalable sizing
    • img { width: 100%; }
  • clamp() — fluid typography with min/max control
    • h1 { font-size: clamp(2rem, 5vw, 4rem); }
  • mobile-first approach — start small, scale up
    • @media (min-width: 768px) { ... }
